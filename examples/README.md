# TabClaw — 示例数据集与体验指南

将以下任意 CSV 文件上传到 TabClaw，直接复制对应的指令粘贴到聊天框即可体验。

---

## 数据集一览

| 文件 | 行数 | 说明 |
|------|------|------|
| `sales_2023.csv` | ~360 | 2023 全年多区域销售明细（产品/区域/季度/利润） |
| `employees.csv` | 85 | 员工 HR 数据（部门/职级/薪资/绩效） |
| `products.csv` | 15 | 产品目录（价格/库存/评分） |
| `orders.csv` | 200 | 订单流水（可与 products.csv 合并） |
| `survey_nps.csv` | 200 | 用户满意度调研（NPS/国家/功能偏好） |

## Case 1 · 销售数据分析（`sales_2023.csv`）

上传：`sales_2023.csv`

### 基础查询
```
哪个区域的全年总收入最高？列出各区域收入排名。
```
```
按季度统计总收入和利润，并计算各季度利润率。
```
```
找出利润最高的前 5 个产品，并显示其平均折扣率。
```

### 数据操作
```
筛选出 Q4 季度、Electronics 品类的所有记录，
按收入从高到低排序后导出。
```
```
以 region 和 category 为维度做透视表，
汇总每个组合的总收入（revenue）。
```
```
为每条记录计算利润率 profit_margin（profit / revenue * 100），
然后找出利润率低于 20% 的记录。
```

### 深度分析
```
Alice Wang 和 Bob Chen 谁的全年销售业绩更好？
对比他们的总收入、总利润和平均折扣。
```
```
分析各产品在不同季度的销量趋势，
哪些产品呈现明显的季节性规律？
```
```
Enterprise 客户和 SMB 客户的购买行为有什么差异？
（平均订单金额、偏好产品、折扣使用情况）
```

## Case 2 · HR 人才分析（`employees.csv`）

上传：`employees.csv`

### 基础查询
```
统计各部门的平均薪资，按从高到低排序。
```
```
找出绩效分数（performance_score）高于 4.5 的所有员工。
```
```
2023 年之后入职的新员工有哪些？按部门分组统计人数。
```

### 数据操作
```
计算各部门的薪资中位数和标准差，找出薪资差异最大的部门。
```
```
筛选 Engineering 和 Data 部门中 level >= 3 的员工，
显示姓名、职级、薪资和绩效。
```
```
统计各城市的员工人数和平均薪资，找出人才最集中的城市。
```

### 深度分析
```
分析薪资与绩效分数的相关性。高薪员工是否表现更好？
```
```
找出在职年限超过 5 年但薪资低于部门平均水平的员工，
他们可能面临薪资倒挂风险。
```
```
对比男女员工在各部门的薪资差异（gender 列），
是否存在显著的性别薪酬差距？
```

## Case 3 · 跨表关联分析（`products.csv` + `orders.csv`）

上传：`products.csv` 和 `orders.csv`（两个文件都上传）

### 合并与关联
```
将 orders 表和 products 表通过 product_id 连接，
统计每个产品类别（category）的总销售额。
```
```
找出库存最低的 5 个产品（products 表 stock_quantity），
查看它们在 orders 表中的销量是否确实很高。
```
```
合并两张表后，计算每个供应商（supplier）的总销售额，
哪个供应商贡献了最多收入？
```

### 高级操作
```
分析每个渠道（channel）的平均订单金额和总销售额，
Online 渠道和 Reseller 渠道的差异有多大？
```
```
找出所有状态为 Returned 的订单，
统计各产品的退货率（退货订单数 / 总订单数）。
```
```
将 orders 和 products 合并，按 category 和 customer_region
做透视分析，查看哪个地区偏好哪类产品。
```

## Case 4 · 用户调研分析（`survey_nps.csv`）

上传：`survey_nps.csv`

### 基础分析
```
计算整体 NPS（净推荐值）。
推荐者（nps_score >= 9）占比、被动者（7-8）占比、贬损者（0-6）占比各是多少？
```
```
按国家统计平均 NPS 和满意度（satisfaction），
哪个国家的用户最满意？
```
```
统计各功能（top_feature）被选为最佳功能的次数，
用户最看重哪个功能？
```

### 细分分析
```
比较不同使用频率（use_frequency）群体的 NPS 和满意度，
重度用户和轻度用户的体验差异有多大？
```
```
找出 NPS 低于 5 的用户，分析他们最常提到的痛点（main_pain_point）。
```
```
按用户角色（role）分组分析满意度，
哪类用户群体满意度最低？需要重点改进什么？
```

### 计算与导出
```
为每位受访者计算一个"参与度得分"：
使用月数 * 使用频率权重（Daily=5, Several/week=4, Weekly=3, Monthly=2, Rarely=1），
找出参与度最高的前 20 名用户。
```

## Plan Mode 体验建议

开启右下角 **Plan Mode** 开关，然后输入：

```
对销售数据做完整的业务分析：
1. 先了解数据概况
2. 找出表现最好和最差的区域与产品
3. 分析季度趋势
4. 识别高价值客户群体
5. 给出改善建议
```

系统会先生成可编辑的执行计划，你可以调整步骤顺序或内容，再点击执行。

## Memory 体验

在对话中说：

```
请记住我偏好中文输出，并且总是以表格形式展示数据结果。
```

之后刷新页面或开新对话，系统会记住这个偏好。

在 Memory 面板可以查看、编辑或删除记忆项。
也可以通过对话遗忘：
```
忘记我的输出格式偏好
```
